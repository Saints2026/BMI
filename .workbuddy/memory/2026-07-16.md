## BMI 项目国际化修复（feature-ai-fix 分支，commit d815a34）

### 问题根因
1. **中文乱码 & 默认显示"英文"** — I18n.load() 用 getResourceAsStream 加载 properties，但 javac -d . 不复制 .properties 文件到输出目录；运行时 -cp . 的 classpath 上找不到 ui_zh.properties/ui_en.properties → ZH/EN 两个 Properties 为空 → I18n.t() 回退返回 key 本身（如 "login.title"）→看起来像英文
2. **语言切换无效** — LoginView/MainView 中大量匿名 new Label(I18n.t("...")) 无法被 refreshTexts() 更新
3. **编码隐患** — Properties.load(InputStream) 隐式编码，应显式用 InputStreamReader(UTF-8)

### 修复方案
- I18n.java: 改为 InputStreamReader(UTF-8) 显式编码 + 增加 src/ 目录回退路径（classpath 找不到时从源码目录加载）
- BmiApplication.java: 启动时从 SettingController 读取持久化语言偏好，显式设置 AppConfig 语言
- LoginView.java: 登录区匿名 Label 改为命名字段（lblUsername/lblPassword/lblVerifycode/lblLang），refreshTexts() 可刷新
- MainView.java: 顶部栏匿名 Label 改为命名字段（lblTopUser/lblTopLang/lblTopTheme），refreshTexts() 可刷新
- .gitignore: com/ → /com/（仅排除根目录编译产物，不影响 src/com/ 源码）

### 验证结果
- javac 编译零错误
- java 运行 BmiApplication 窗口正常弹出
- 已提交并推送到 feature-ai-fix 分支（commit d815a34）

- .vscode/settings.json 中的 `java.project.sourcePaths` 配置丢失或未生效，导致 VSCode Java Language Server 找不到 src/ 目录下的所有类

### 修复方案
- 更新 .vscode/settings.json，确保 `java.project.sourcePaths: ["src"]` 配置正确
- 保留 `referencedLibraries` 包含 4 个 JUnit jar + 4 个 JavaFX SDK jar（外部路径 D:/javafx-sdk-24.0.1/lib）
- 移除之前的 `java.project.outputPath: "bin"` 配置（按用户指定模板）

### 验证结果
- javac 全量编译零错误（41 个 class 文件全部生成）
- TestAiService.class 编译成功，所有 import 可解析
