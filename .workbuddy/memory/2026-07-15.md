# 2026-07-15 工作日志

## BMI 项目编译错误修复（feature-ai-fix 分支）

### 问题根因
1. **JavaFX 依赖缺失** — lib/ 中放了 4 个空 stub jar（javafx-*-23.0.2.jar，uncompressed size 0），.vscode/settings.json 错误引用了这些 stub
2. **JUnit 依赖无效** — lib/junit-jupiter-5.10.0.jar 是 Maven 聚合 jar（仅含 module-info），不含实际 API 类
3. **AiService 已删除但仍被引用** — feature-ai-fix 分支将 AI 逻辑从 AiService 迁移到 AiHealthClient，但 BmiApplication.java 仍引用已删除的 AiService
4. **BodyRecord 类型不匹配** — AiController/AiHealthClient 用 com.bmi.model.ai.BodyRecord（简化 DTO），ReportController 用 com.bmi.model.BodyRecord（完整实体），调用时类型不匹配
5. **ChartView 缺少 HBox 导入**，BodyFatEstimatorTest 缺少 BodyFatEstimator 导入
6. **TestAiService 硬编码 API Key**（违反宪章第 4/7 节）

### 修复方案
- .vscode/settings.json：引用外部 JavaFX SDK 24.0.1（D:/javafx-sdk-24.0.1/lib）+ 全部 JUnit jar
- lib/：删除 4 个空 stub jar，下载 junit-jupiter-api-5.10.0.jar + 3 个依赖 jar
- BmiApplication：移除 AiService 引用，新增 createAiController() 从 ai-key.properties 读取密钥
- AiController/AiHealthClient：统一使用 com.bmi.model.BodyRecord
- AiHealthClient.buildJsonBody：getMeasureDate() → getMeasureTime()
- TestAiService：移除硬编码 Key，改为从 ai-key.properties 读取
- ChartView：补充 HBox 导入；BodyFatEstimatorTest：补充 BodyFatEstimator 导入
- 新增 ai-key.properties.example 模板；.gitignore 允许 .vscode/settings.json

### 验证结果
- javac 编译主源码 + 测试文件：零错误
- java 运行 BmiApplication：JavaFX 窗口成功弹出（进程稳定运行）
- lib/ 仅含 5 个 JUnit jar，无违规文件
- 已提交并推送到 feature-ai-fix 分支（commit 969c0f2）

### 第二轮修复：VSCode BodyFatEstimator 红线问题（commit 5a0ac5a）
- **根本原因**：lib/junit-jupiter-5.10.0.jar 是空聚合器（仅含 module-info.class），与 junit-jupiter-api 的 module-info 产生 JPMS 模块系统冲突，导致 VSCode Java Language Server 项目模型构建失败，所有源码类无法解析
- **修复**：删除空聚合器 jar，从 settings.json 移除其引用，添加 outputPath 配置
- javac 编译验证：仍然正常（BodyFatEstimator + BodyFatEstimatorTest 零错误）
- 用户需要执行 VSCode Java: Clean Java Language Server Workspace 后重新加载
