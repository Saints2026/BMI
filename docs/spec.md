## 2. 产品概述与目标用户

- **一句话定位**：一款运行在桌面端的轻量级个人体质评估工具，录入身高体重即可计算 BMI、预测体脂率、追踪历史趋势并获取 AI 健康建议。
- **目标用户画像**：关注自身体质健康、希望以低成本方式持续了解体重与体脂变化走向的普通用户（如上班族、健身初学者、体重管理者）。用户具备基本电脑操作能力，无需专业医学知识，期望界面直观、反馈即时、建议可读。

## 4. 功能需求

### FR-01 用户登录与注册

- **功能描述**：提供账号注册、账号密码登录、登录态校验功能。密码需满足强度规则。
- **输入**：
  - 注册：`username`（3–20 位，字母/数字/下划线）、`password`（见密码规则）、`confirmPassword`。
  - 登录：`username`、`password`。
- **密码校验规则**：
  - 长度 8–20 位；
  - 至少包含大写字母、小写字母、数字中的两种；
  - 不允许与用户名相同；
  - 两次输入需一致（注册时）。
- **处理逻辑**：
  - 注册：校验用户名唯一性（查 `t_user`），校验密码规则，对密码做不可逆散列（如 SHA-256 + 随机盐）后写入数据库。
  - 登录：按 `username` 查询并比对散列值，校验通过后在内存中维护会话用户对象（不依赖 Web Session）。
- **输出**：登录成功进入主界面；注册成功提示并跳转登录；失败给出明确原因。
- **异常处理**：
  - 用户名已存在 → 提示「该用户名已被注册」。
  - 密码不符合规则 → 提示具体未满足项（如「密码至少包含两种字符类型」）。
  - 用户名/密码错误 → 提示「用户名或密码错误」（不泄露具体哪一项）。

### FR-02 身高体重录入

- **功能描述**：在主界面录入单次身高与体重数值并保存为一条待计算记录。
- **输入**：`heightCm`（数值，单位 cm）、`weightKg`（数值，单位 kg），可选 `measureTime`（默认当前时间）。
- **合法区间校验**：
  - 身高：`[50, 250]` cm；
  - 体重：`[10, 300]` kg；
  - 必须为正数，非数值或空值拒绝。
- **处理逻辑**：在视图层做格式与区间校验，校验通过后交由控制器调用 `Record` 业务方法组装记录（暂不持久化直到 FR-03 计算完成一并落库）。
- **输出**：校验通过则进入 BMI 计算流程；校验失败在输入框旁给出明确错误提示。
- **异常处理**：
  - 超出区间 → 提示「身高应在 50–250cm 之间」/「体重应在 10–300kg 之间」。
  - 非数字 → 提示「请输入有效数字」。

### FR-03 BMI 计算与分级

- **功能描述**：依据身高体重计算 BMI，并按中国成人标准给出分级。
- **输入**：`heightCm`、`weightKg`（来自 FR-02）。
- **处理逻辑**：
  - 公式：`BMI = weightKg / (heightM)²`，其中 `heightM = heightCm / 100.0`。
  - 计算保留 1 位小数（四舍五入）。
  - 分级（中国成人标准）：
    - 偏瘦：`BMI < 18.5`
    - 正常：`18.5 ≤ BMI ≤ 23.9`
    - 超重：`24.0 ≤ BMI ≤ 27.9`
    - 肥胖：`BMI ≥ 28.0`
- **输出**：BMI 数值（1 位小数）与分级标签，显示于主界面，并随记录一并入库。
- **异常处理**：输入非法（被 FR-02 拦截）时不进入计算，提示沿用 FR-02 文案。

### FR-04 体脂预测

- **功能描述**：基于 BMI、年龄、性别估算体脂率。
- **输入**：`bmi`（来自 FR-03）、`age`（整数，`[1, 120]`）、`gender`（枚举：男=1 / 女=0）。
- **处理逻辑**：采用 Deurenberg 公式：
  - `体脂率% = 1.2 × BMI + 0.23 × age − 10.8 × gender(男1/女0) − 5.4`
  - 结果保留 1 位小数。
- **输出**：体脂率数值（1 位小数），显示并随记录入库。
- **异常处理**：
  - 年龄超界 → 提示「年龄应在 1–120 之间」。
  - 性别未选择 → 提示「请选择性别」。

### FR-05 历史记录管理

- **功能描述**：持久化每次测量，并支持按时间查询与删除。
- **输入**：查询条件（起止时间可选）、待删除记录 ID。
- **处理逻辑**：
  - 落库：每次 FR-02~FR-04 完成后，将 `measure_time, height, weight, bmi, body_fat, user_id` 写入 `t_record`。
  - 查询：按 `user_id` + 时间区间检索，默认按时间升序返回。
  - 删除：按记录主键删除单条，仅允许删除当前用户自己的记录。
- **输出**：记录列表（时间/身高/体重/BMI/体脂率）；删除后列表刷新。
- **异常处理**：
  - 无记录 → 提示「暂无历史记录」。
  - 删除失败/越权 → 提示「删除失败，请重试」。

### FR-06 动态折线图表

- **功能描述**：以时间为横轴，绘制 BMI/体重/体脂率趋势折线图，支持指标切换。
- **输入**：当前用户的历史记录集合；当前选中的指标（BMI / 体重 / 体脂率）。
- **处理逻辑**：
  - 使用 Swing 自绘（`JPanel` + `Graphics2D`）或 JFreeChart（jar 入 `lib/`，二选一）绘制。
  - 横轴：测量时间；纵轴：所选指标数值；每个数据点对应一条 `t_record`。
  - 提供指标切换控件（如下拉框/单选按钮）即时重绘。
- **输出**：折线图面板，随指标切换刷新。
- **异常处理**：
  - 数据点 < 2 → 不绘制趋势线，提示「至少需 2 条记录才能显示趋势」。
  - 无记录 → 显示空图区与提示文案。

### FR-07 AI 健康建议

- **功能描述**：将用户最新指标与近期历史趋势发送给 AI 接口，返回饮食/运动/健康建议文本。
- **输入**：最新指标（BMI、体脂率、体重、年龄、性别）+ 近 N 条历史摘要（趋势描述）+ 系统提示词。
- **处理逻辑**：
  - 经 `model.ai` 层 `XxxClient` 使用原生 `HttpURLConnection` 组装 JSON 请求体，调用外部大模型接口。
  - API 密钥**仅**从 `ai-key.properties` 读取，禁止硬编码。
  - 设置连接超时与读取超时；成功则解析返回文本。
- **输出**：建议文本，分段展示「饮食/运动/健康」要点。
- **异常处理**：
  - 请求超时（>10s）→ 中断并提示「AI 建议请求超时，请稍后重试」。
  - 网络/接口错误 → 降级提示「暂时无法获取 AI 建议，请检查网络或稍后再试」。
  - 密钥缺失 → 提示「AI 服务未配置，请联系管理员」。

## 6. 非功能需求

- **性能**
  - 历史查询（千条级）响应 < 1s；折线图重绘 < 500ms；AI 请求超时阈值 10s。
  - 桌面端启动后主界面首屏 < 3s（普通办公机）。
- **易用性**
  - 全中文界面，关键操作有即时校验与明确提示文案。
  - 录入→计算→结果展示在单次交互内完成，无需多窗口跳转。
  - 支持键盘与鼠标操作，错误输入在输入框旁就地提示。
- **数据安全**
  - API 密钥**仅**读取 `ai-key.properties`，数据库配置**仅**读取 `db-config.properties`，**禁止硬编码**于源码（两文件已 gitignore）。
  - 用户密码以「随机盐 + SHA-256」不可逆散列存储，不保存明文。
  - 数据访问限于 `model.db` 层 DAO；视图不直接访问数据库或 AI。
  - 记录按 `user_id` 隔离，禁止越权读取/删除他人数据。
- **技术约束（来自宪章）**
  - 语言 Java 8+；架构自研 MVC（controller/model/view）；视图 Swing；数据 JDBC + MySQL 或 SQLite；AI 用 `HttpURLConnection` 原生调用；第三方 jar 置于 `lib/`（无 Maven/Gradle）。
